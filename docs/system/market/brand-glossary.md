# uTap Brand Glossary

The single page that answers: "How do we write that name?"

If a word appears here, use it exactly as shown. If a word is missing and you need it in copy, add it in the same PR.

---

## Product names

| Term | Correct | Wrong | Notes |
|------|---------|-------|-------|
| The brand | **uTap** | UTAP, utap, U-Tap, U Tap | Lowercase `u`, capital `T`, no hyphen, no space. |
| Shopping module | **uShop** | Ushop, U-Shop, ushop, USHOP | Lowercase `u`, capital `S`. |
| Events module | **uGig** | Ugig, U-Gig, ugig | Lowercase `u`, capital `G`. |
| Vendor portal | **uTap for Vendors** or **the vendor portal** | "the vendor app", "Vendor Hub", "Vendor Console" | Vendors don't see "portal" everywhere — see vendor copy doc. |
| Admin console | **uTap Admin** | "Backoffice", "Admin Hub" | Internal only — students never see this name. |
| Marketing site | **utap.co.za** | "the website", "marketing site" | Use the URL in public copy. |

### When in doubt

> Always say **uTap** in marketing. Inside the product, drop to lowercase only when it's a verb in a sentence: "Tap into the library" — not "uTap into the library".

---

## Module language

**uShop** and **uGig** are *parts of* the uTap mobile app, not separate products. Don't write "Download uShop" — there's no such download. Write "Open uShop in uTap" or "Find it in uTap's uShop tab".

## Money

- Currency symbol: **`R`** followed by a thin space, then the number.
- Whole rand: `R 45`, not `R45.00`.
- Cents: only when meaningful — `R 12.50`, never `R 12.5`.
- Large numbers: thin space thousands. `R 12 500` (not `R12,500`).
- Free: `Free` (capital F when standalone, lowercase in the middle of a sentence).

## Roles

| Role | Public copy | Internal copy |
|------|-------------|----------------|
| Student | "you" / "student" | "student" |
| Vendor | "you" / "your store" | "vendor" |
| Vendor staff | "team member" | "vendor staff" |
| Admin | — never appears in student copy | "admin", "uTap team" |

In student-facing copy, **the student is always "you"**.

In vendor-facing copy, **the vendor is "you"** and their team is "your team".

## Verbs we own

These verbs should keep their meaning across the product. Don't reassign.

| Verb | Means | Example |
|------|-------|---------|
| **Tap** | Place phone near a reader | "Tap to enter the library" |
| **Scan** | Read a card with the phone | "Scan your student card" |
| **Save** | Persist a card to your wallet | "Save card to uTap" |
| **Order** | Buy from a vendor for pickup | "Order from Campus Café" |
| **Pick up** (verb) / **Pickup** (noun) | Collect an order | "Pick up your order" / "Pickup ready" |
| **Pay** | Send money via Yoco | "Pay R 45 with Yoco" |
| **Open** (a store) | Vendor lists a store | "Open my store" |
| **List** (a product) | Vendor adds a product | "List a product" |
| **Approve / Suspend / Reinstate** | Admin actions on vendors | Internal only |

## Capitalisation rules

| Element | Style | Example |
|---------|-------|---------|
| Page titles | Sentence case | "Order details" |
| Buttons | Sentence case | "Place order" |
| Tab labels | Sentence case | "Orders" |
| Section headers in marketing | Sentence case (occasional Title Case if the brand voice calls for it on a hero) | "Skip the queue" |
| Modal titles | Sentence case | "Cancel this order?" |
| App store title | "uTap" | not "UTAP - Campus Wallet" |
| Email subjects | Sentence case | "Your order is ready" |

Never SHOUT in all caps. Don't use Title Case For Every Word.

## Common phrasing decisions

| Two options | We use | Why |
|-------------|--------|-----|
| "Sign up" vs "Register" | **Sign up** | Simpler, more verbal |
| "Sign in" vs "Log in" | **Sign in** | Matches sign-up |
| "Log out" vs "Sign out" | **Sign out** | Consistent |
| "Cart" vs "Bag" vs "Basket" | **Bag** | Reads natural on mobile |
| "Email" vs "Email address" | **Email** | One word is enough |
| "Phone" vs "Mobile number" | **Phone** | Same reason |
| "Settings" vs "Preferences" | **Settings** | Universally understood |
| "Help" vs "Support" vs "FAQ" | **Help** | Friendlier |
| "Contact us" vs "Get in touch" | **Get in touch** | Warmer |
| "Save" vs "Confirm" vs "Done" | Context-specific: **Save** for forms, **Confirm** when something's about to happen, **Done** for read-only moments | |

## Tone of named features

A few features carry a specific micro-voice:

- **uTap wallet** — protective, warm. "Your card is safe with uTap."
- **uShop** — peppy, sensory. "Hot chips, just round the corner."
- **uGig** — energetic, social. "Game day, sorted."
- **Vendor portal** — calm, operational. "3 new orders to confirm."
- **Admin** — neutral, precise. No vibe.

Don't import the uGig tone into a vendor finance page, and don't import the vendor calmness into a student push notification.

---

## Edge cases

- **uTap on a poster in all caps:** if a design absolutely demands all-caps treatment, the brand stays **UTAP** (no spaces, no hyphen). Use only when the rest of the headline is also all-caps. Never **U-TAP** or **UTAP.**.
- **uTap as a verb:** discouraged in marketing but allowed in informal social posts. Cap it as **uTap into** (not "to uTap").
- **Plural:** we don't pluralise the brand. "Two uTaps" is wrong. Say "two uTap accounts" or "two phones".

## Maintenance

When adding a new feature name:

1. Decide the user-facing name with Marketing.
2. Add a row to this file.
3. Add it to [`voice-guidelines.md`](voice-guidelines.md) banned/preferred tables if it overlaps a word we already use.
4. Update the matching copy spec.
