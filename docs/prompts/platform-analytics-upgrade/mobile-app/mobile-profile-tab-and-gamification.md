# Prompt 2 of 3 — Student App: Profile Tab (gamified stats, badges, streaks & monthly wrap)
> Run after Prompt 1 is merged into Supabase.

---

Enhance the Profile tab in the utap-apps React Native app using data from the `student_gamification` table and `badges` table. Everything lives inside the existing Profile tab — do not add a new tab. The experience should feel personal, fun, and motivating — like a student's campus identity card came to life.

1. PROFILE HEADER ENHANCEMENT
Keep the existing avatar initials, name, email, university, and location. Add directly below the existing header:
- Campus rank chip: "🏆 #12 on campus" — pull from campus_rank
- Campus percentile: "Top 8% at [University Name]" — shown as a small subtitle line
- Current streak badge: "🔥 5-day streak" — use an animated flame icon if the streak is 3+ days (React Native Animated API, simple pulse or glow effect)
- Total savings pill: "💰 Saved R340 with uTap deals"

Keep this section compact — it sits between the existing header and the rest of the profile content. Do not push existing shortcuts (Orders, Cards) out of view.

2. BADGES SECTION
Add a "Your Badges" horizontal scroll section below the header area:
- Show earned badges as circular icon cards with the badge colour, icon, and name
- Locked/unearned badges shown as grayed-out with a lock icon and progress hint: "8/10 orders" toward the next threshold
- Tapping an earned badge opens a small bottom sheet with the badge name, description, and the date it was earned
- Tapping a locked badge shows what's needed to unlock it
- Show a "X badges earned" count above the scroll row
- Order: earned badges first (sorted by date earned desc), then locked badges sorted by proximity to unlock

3. SPENDING STATS SECTION
Add a "Your Stats" card section (use React Native Paper Card or a custom styled card, consistent with the existing app style):
- Most purchased item: product name + a "Reorder" button that deep-links to that product in Ushop
- Favourite store: store name + "Visit" button that navigates to the store detail screen
- Total orders all time
- Total spent all time (ZAR)
- This month vs last month: "You've spent R280 this month — R120 less than last month 📉" or "R150 more than last month 📈" — use a simple coloured trend indicator, not a chart

4. MONTHLY WRAP (end-of-month feature)
Implement a "Monthly Wrap" card that appears in the Profile tab:
- Show automatically during the last 3 days of the month AND the first 3 days of the next month
- Content: favourite meal this month, most visited store, total spent, total orders, biggest single order, how much saved via promotions, a fun generated headline like "You kept [Store Name] in business this month 😄"
- The headline should be generated by picking from a set of ~10 fun template strings based on the student's stats — hardcode the templates in the app, no AI call needed
- Style it as a distinct full-width card with a gradient background (use the app's existing colour tokens) — it should feel like a "special moment", not a regular stat card
- Include a share button that uses React Native Share API to share a plain-text summary: "My uTap November Wrap: X orders, R Y spent, favourite store: Z 🎓"

5. LEADERBOARD TEASER
Add a compact "Campus Leaderboard" section at the bottom of the Profile tab stats area:
- Show the top 3 students on campus by total orders this month as anonymous entries: "🥇 A student on campus — 47 orders", "🥈 Another student — 38 orders"
- Show the current student's own rank highlighted: "You — #12 — 19 orders"
- IMPORTANT: Never show other students' real names or identifiable info — show "A student at [University]" or similar anonymised label
- A "See full leaderboard" button that opens a modal with top 20, same anonymisation rules, current student's row always visible and highlighted

Be proactive: use the React Native Animated API for the streak flame and badge unlock animations — keep them subtle (scale + opacity, 300ms). Follow existing screen layout patterns in the Profile tab. If React Native Paper doesn't support a component naturally, use a plain View/Text/TouchableOpacity — consistency with the existing codebase matters more than novelty. Fetch all gamification data in a single Supabase query on Profile tab focus, cache with React Context or a simple hook.
