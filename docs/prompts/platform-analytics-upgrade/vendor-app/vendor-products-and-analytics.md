# Prompt 3 of 3 — Vendor: Products Page Analytics (performance, dead stock & insights)
> Run after Prompt 2 is working.

---

We need to add product analytics to the Products page in the vendor dashboard (utap-vendors). The data source is the `vendor_product_stats` table created in the earlier migration.

Here is what I need:

1. ANALYTICS SUMMARY BAR
At the top of the Products page, add a row of KPI metric cards (reuse the existing dashboard card component if one exists). Show:
- Total products listed
- Total units sold (all time)
- Dead stock count (products with is_dead_stock = true) — highlight in amber if > 0
- Top product this month (name + units sold)

2. DEAD STOCK WARNING PANEL
If the vendor has any products where is_dead_stock = true, show a collapsible warning section above the product table with the heading "Products needing attention". List each dead stock item with:
- Product name + image thumbnail
- Days since last sale
- Current stock level
- A quick-action button: "Create discount" — this should either open the existing promotions/voucher flow or link directly to it with the product pre-selected

3. PRODUCTS TABLE ENHANCEMENTS
Enhance the existing products table (do not replace it — add to it) with two new columns:
- Units sold (from vendor_product_stats, show 0 if no data yet)
- Performance badge: "Top seller" (top 20% by revenue), "Steady" (has sales, not top), "Slow" (1–2 sales in 30 days), "Dead stock" (is_dead_stock = true)

Keep all existing columns and actions — only add these two.

4. PRODUCT DETAIL ANALYTICS (on existing edit/view flow)
When a vendor views or edits a product, add an analytics section below the form with:
- A bar chart (recharts) showing units sold per week for the last 8 weeks
- Total revenue generated by this product
- "Frequently bought with" — top 2 products purchased in the same order (compute from orders data or vendor_product_stats if you can derive it)
- Estimated stock-out date: based on average daily sales over last 7 days, calculate "At current pace, stock runs out in ~X days" — show as a plain text stat, not a chart

5. PEAK HOUR INSIGHT (optional enhancement — implement if straightforward)
On the main products or dashboard overview page, add a small heatmap or bar chart showing which hours of the day generate the most orders for this vendor. Use order created_at timestamps grouped by hour. This helps vendors know when to be fully stocked and staffed.

Be proactive: use recharts consistently (it is already a dependency). Follow existing component patterns in the codebase. If a chart or stat would require a disproportionate amount of new infrastructure for little value, flag it with a short comment and skip it — prioritise the dead stock warning and the product performance badges as the highest-value additions.
